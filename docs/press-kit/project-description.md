## 500 words

> Webarmonium is a networked musical instrument that runs in the browser and plays the web. It is a single artwork with two modes of operation.
>
> **The landing page** opens onto an algorithmic composition that listens to three open public data streams in real time: Wikipedia's Recent Changes feed, HackerNews's new stories, and GitHub's Public Events. Each event the work hears is translated, through a fixed mapping, into a note. Horizontal position determines frequency in a 220–880 Hz band; vertical position modulates harmonic content and amplitude; the data source selects timbre. A live ticker on the page surfaces the specific web event that caused each note ("Napoleon edited · C#4", "ycombinator/show · GitHub · A4"), making the data → sound causality legible. The composition that emerges is the unconscious performance of millions of strangers who will never hear it.
>
> **Collaborative rooms** invite up to four remote performers into a shared browser session where they compose together intentionally. They share gestural input (tap, hold, drag), a programmable step sequencer of 3–16 steps, three fully synthesized drum kits (808, Acoustic, Electronic—no sample files), and a sound-design panel of eight synth presets with per-voice controls. An audition mode generates phrases autonomously using PHI-based interval timing, so the instrument continues to perform itself when no one is touching it. All inputs feed an algorithmic composition engine that models musical form, voice leading, and twenty genre profiles. The room maintains a 24-hour rolling memory of interaction patterns that shapes its tonal character over time.
>
> The mapping is deterministic throughout. There is no randomness, only the structured translation of human activity—conscious and unconscious, programmatic and gestural—into sound. The same network event, the same gesture, the same room state will always produce the same musical output.
>
> Webarmonium is interested in what music sounds like when authorship dissolves into network. The traditional musical instrument translates one performer's gesture into sound. Webarmonium replaces that single gesture with a distribution: across millions of anonymous online gestures, across a small handful of people who happen to share a browser session at any given moment, across autonomous algorithms that perform when humans walk away. What we call musical authorship is here distributed across data streams, anonymous strangers, autonomous algorithms, and the few performers who happen to share a browser session.
>
> The work is built in vanilla JavaScript with the Web Audio API, Tone.js, Socket.io, and PixiJS, and was developed entirely through iterative dialogue with a large language model. It exists only at webarmonium.net—it is not packaged, not downloadable, not installable. It runs continuously, audible to anyone with a browser. There is no admission fee, no account, no purchase.
>
> Webarmonium is a single, persistent, public artwork.
>
> Live at webarmonium.net.
